# Agent Documents

This directory contains durable guidance for coding agents working on gitlode.

These documents are not product design contracts and are not temporary handoff notes. They describe
collaboration patterns, review workflows, and agent-facing working principles that should remain
useful across tasks.

Agent documents may explain how agents should work with humans or navigate repository
documentation, but they must not become independent sources of truth for gitlode behavior. Product
and implementation contracts still belong in the canonical documents listed from the repository
`AGENTS.md` and the package documentation index.

## Documents

- [`collaborative-work.md`](collaborative-work.md): general collaboration principles for LLM agents
  and human users, especially for iterative engineering tasks.
